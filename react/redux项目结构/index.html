react与redux

发表于 2020-05-20 分类于 react
本文字数： 985 阅读时长 ≈ 1 分钟

在上手react之前，一直拿vue与react进行比较，不管是框架本身还是其全家桶；一开始是从react基础开始的时候，并没有遇到什么困难，只是react多了jsx语法，其他并无差别，再后来开始研究redux的时候，也对比着vuex进行学习，但是后来，发现两者并不相同，但都遵循了单向数据流的模式，redux由 store 、action-creator、reducer 、actionType组成，当用户行为dispatch相应的action-creator的时候，把相应的动作行为传给reducer，reducer根据不同的type返回新的state；但是在vuex当中是不同的，vuex是由state、getter、mutations、action组成，若需更改state，需要使用mutaions来进行更改，也是唯一更改state的方式，而actions是需要提交mutations，让mutaions执行更改state的任务，其中mutations是同步的，而actions是异步的。两者在项目搭配使用的时候，效果相差很大。

在vue搭配vuex进行项目开发的时候，是分的很开的，耦合不太厉害，但是在react搭配redux进行项目开发的时候，耦合就很严重了，以至于不太习惯这种开发模式

了解完redux基础概念以后，尝试向react项目当中添加redux，为了项目壮大以后的扩展性考虑，怎么组织redux结构呢？以下来进行梳理一下思路:

首先安装redux

再安装react-redux

  • react-redux 提供了两个重要组件:Provider connect ; 其中Provider是提供对应的store给其所有子组件，connect是将mapstateToProps、mapdispatchToProps等信息传递到指定组件，也是连接redux与react的桥梁。

接下来,组织项目的目录结构，redux融合在react更加和谐，并且更加易用
尝试了几种组织方式: 按照类型的方式 、 按照功能方式、Ducks
Ducks是以应用状态作为模块的划分依据，并且将reducer、action Types、action都写在同一个文件当中，实现了低耦合并且相对独立，冗余无用的文件也不会产生。
